Wrong email client opening? Configure your browser

If the wrong app opens (or nothing happens at all), check both system and browser settings:

macOS system settings (required if nothing opens)
  1. Default email reader: Open the Mail app → Mail → Settings → General → set Default email reader to your preferred app (Mail, Outlook, etc.)
  2. For Gmail in Chrome: Also set Chrome as default browser: System Settings → Desktop & Dock → Default web browser → Chrome. macOS routes mailto through the default browser.
Chrome
  1. Settings → Privacy and security → Site settings → Additional permissions → Handlers
  2. Turn on Allow sites to ask to become default handlers for protocols
  3. Open your webmail: Gmail, Outlook.com, Yahoo Mail, etc.
  4. Click the protocol handler icon (⊞ in the address bar) and choose Allow / Use Gmail
  5. Important: Go to chrome://settings/handlers and set Gmail (or your webmail) as the default handler for mailto. If no default is set, Chrome may do nothing when mailto links are clicked.
Edge
  1. Go to edge://settings/content/handlers
  2. Enable Allow sites to ask to become default handlers for protocols
  3. Open your webmail (Gmail, Outlook.com, Yahoo Mail, etc.), then click the handler icon in the address bar and allow it
  4. Important: Set your webmail as the default handler for mailto in the handlers list — otherwise mailto links may do nothing.
Firefox
  1. Settings → General → scroll to Applications
  2. Find mailto and set it to your preferred handler (e.g. Gmail, Outlook.com)
Safari (Mac)
  1. Safari has no built-in mailto handler picker. Use the Gmail Mailto Handler extension, or open this page in Chrome/Firefox and set Gmail there
  2. Windows: Settings → Apps → Default apps → Email to choose your default email app

After changing handlers, refresh this page and try sending again.
